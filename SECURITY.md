# 安全政策

## 支持范围

本项目仍处于早期原型阶段。安全修复默认面向当前 `main` 分支；旧提交和未发布实验分支不承诺长期维护。

## 漏洞报告

不要为疑似漏洞、暴露密钥、成员数据泄露或可利用攻击路径创建公开 issue。请优先使用 GitHub private security advisory；如果仓库暂未启用该功能，请使用仓库所有者列出的私有维护者渠道。

报告时请尽量包含：

- 受影响的 commit、版本或部署环境
- 可复现步骤
- 预期影响和实际影响
- 是否涉及凭据、token、成员数据、数据库导出或生产日志
- 已经采取的临时缓解措施

在维护者确认和修复前，请不要公开 PoC、截图中的敏感数据或可直接复用的攻击细节。

## 处理流程

维护者收到报告后会先确认影响范围，再决定修复优先级、是否需要临时缓解说明以及是否需要发布安全公告。修复完成前，相关讨论应保持在私有渠道中。

## 部署安全提醒

默认配置只面向本地开发。任何非本地部署都必须设置 `BIG_APPLE_ENV=production` 或 `DJANGO_ENV=production`，提供真实的 `DJANGO_SECRET_KEY`，关闭 debug，配置 `ALLOWED_HOSTS`，并启用安全 cookie、HTTPS redirect 和 HSTS。

不要提交 `.env`、数据库导出、成员数据、生产日志、生成缓存或 `output/` 产物。
